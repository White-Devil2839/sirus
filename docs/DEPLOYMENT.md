# Deployment

Free-tier stack: **Vercel** (client) + **Render** (API) + **MongoDB Atlas** (M0).

## 1. Database — MongoDB Atlas
1. Create a free M0 cluster → Database Access: create a user → Network Access: allow
   `0.0.0.0/0` (or Render's IPs).
2. Copy the SRV connection string as `MONGODB_URI`.

## 2. API — Render
The repo ships a blueprint: `server/render.yaml`.

1. Render → New → Blueprint → point at the repo.
2. Set env vars: `MONGODB_URI`, `CLIENT_ORIGIN` (your Vercel URL), optional
   `GROQ_API_KEY`, `DEEPGRAM_API_KEY`. `JWT_SECRET` is generated by Render.
3. Health check: `/api/health`. Free instances cold-start (~30–50 s) after idle.
4. Seed once from your machine:
   `MONGODB_URI='<atlas-uri>' npm run seed` (from `server/`).

## 3. Client — Vercel
1. Import the repo → Root Directory `client` → framework Vite (auto).
2. Env var: `VITE_API_URL = https://<your-render-service>.onrender.com/api`.
3. SPA rewrites are handled by `client/vercel.json`.

## 4. After deploy
- Update `CLIENT_ORIGIN` on Render to the final Vercel URL (CORS is pinned to it).
- Verify: `/api/health` returns `ok`, landing page renders the sample library, demo
  logins work, admin Generate produces a report (live if keys set, seeded otherwise).

## CI
`.github/workflows/ci.yml` runs on every push/PR: client build + server test suite
(in-memory Mongo — no secrets needed) + `npm audit --audit-level=high` as a gate.
